# 個人網站 — 功能規格（Spec）

**Date:** 2026-03-14
**Status:** Approved

---

## 1. 目標與範圍

打造一個個人網站，包含：
- **個人資訊頁**：自我介紹、技能標籤、聯絡方式
- **部落格**：學習筆記，以 Markdown 檔案管理

不在此次範圍內（未來可擴充）：
- Projects / 作品集區塊
- 雙語（中英文切換）
- 留言或互動功能

---

## 2. 頁面清單

| 路徑 | 頁面名稱 | 說明 |
|------|----------|------|
| `/` | Landing Page | 網站入口，展示名字與簡短自我描述 |
| `/home` | 主頁 | 個人資訊（About、Skills、Contact）單頁滾動 |
| `/blog` | 文章列表 | 所有文章的標題與日期 |
| `/blog/:slug` | 單篇文章 | 完整文章內容 |

---

## 3. 頁面功能規格

### 3.1 Landing Page（`/`）
- 全螢幕版面，不顯示 Navbar
- 顯示使用者名字
- 打字機動畫，循環顯示以下描述文字：
  - `前端工程師`
  - `學習筆記愛好者`
  - `Lifelong Learner`
- 提供 GitHub 連結（圖示）
- 提供「進入網站」按鈕，點擊後導向 `/home`

### 3.2 Navbar（共用）
- 出現在 `/home`、`/blog`、`/blog/:slug` 頁面
- 提供導覽連結：首頁（`/home`）、部落格（`/blog`）

### 3.3 About 區塊（`/home`）
- 顯示使用者姓名
- 顯示一段自我介紹文字

### 3.4 Skills 區塊（`/home`）
- 顯示技能標籤列表

### 3.5 Contact 區塊（`/home`）
- 顯示 GitHub 連結
- 顯示 Email 連結

### 3.6 文章列表（`/blog`）
- 列出所有文章，每篇顯示：標題、日期
- 點擊任一文章可進入該篇文章頁面
- 若無文章，顯示「目前沒有文章」提示

### 3.7 單篇文章（`/blog/:slug`）
- 顯示文章標題、日期、內文
- Slug 對應文章檔名（例如 `react-hooks-intro.md` → `/blog/react-hooks-intro`）
- 若 slug 無對應文章，顯示「找不到文章」訊息

---

## 4. 文章格式（使用者角度）

文章以 Markdown 格式撰寫，檔案需含以下欄位：

```yaml
---
title: 文章標題
date: 2026-03-14
---

文章內容
```

---

## 5. 不在範圍內

- 搜尋功能
- 文章標籤 / 分類
- 深色 / 淺色主題切換
- SEO / meta tags
- 部署設定
- Projects / 作品集
- 雙語切換
- 留言功能
